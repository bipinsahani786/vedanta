## Local Development Setup

### Prerequisites
- PHP 8.3+
- Composer
- Node.js 18+ & npm
- MySQL 8.0+

### Step-by-step

```bash
# 1. Clone the repository
git clone <your-repo-url>
cd Vedanta

# 2. Install PHP dependencies
composer install

# 3. Install Node dependencies
npm install

# 4. Copy environment file
copy .env.example .env

# 5. Generate application key
php artisan key:generate

# 6. Configure your .env file (see section below)
# Edit DB_DATABASE, DB_USERNAME, DB_PASSWORD, MAIL_*, etc.

# 7. Run database migrations
php artisan migrate

# 8. Create storage symlink (for uploaded files)
php artisan storage:link

# 9. Build frontend assets
npm run build

# 10. Start the development server
php artisan serve

# 11. (Optional) Run Vite for hot-reload during development
npm run dev
```

Visit: **http://127.0.0.1:8000**

### Default Admin Login
Create the first admin by seeding or manually inserting into the `users` table with `role = 'admin'`.

## Environment (.env) Configuration

Below is a complete `.env` template with all variables explained:

```dotenv
# ============================================
# APP SETTINGS
# ============================================
APP_NAME="Vedanta Placement Agency"
APP_ENV=production           # Use 'local' for development
APP_KEY=                     # Auto-generated by: php artisan key:generate
APP_DEBUG=false              # MUST be false in production
APP_URL=https://yourdomain.com

# ============================================
# DATABASE
# ============================================
DB_CONNECTION=mysql
DB_HOST=127.0.0.1            # Hostinger: use 'localhost'
DB_PORT=3306
DB_DATABASE=your_db_name
DB_USERNAME=your_db_user
DB_PASSWORD=your_db_password

# ============================================
# SESSION & CACHE
# ============================================
SESSION_DRIVER=database
SESSION_LIFETIME=120
CACHE_STORE=database
QUEUE_CONNECTION=database    # Use 'sync' if queue workers are not available

# ============================================
# MAIL (Hostinger SMTP — see Email Setup below)
# ============================================
MAIL_MAILER=smtp
MAIL_SCHEME=ssl
MAIL_HOST=mail.yourdomain.com
MAIL_PORT=465
MAIL_USERNAME=info@vedantaplacementagency.in
MAIL_PASSWORD=your_email_password
MAIL_FROM_ADDRESS=info@vedantaplacementagency.in
MAIL_FROM_NAME="Vedanta Placement Agency"

# ============================================
# PHONEPE (Payment Gateway)
# ============================================
PHONEPE_MERCHANT_ID=your_merchant_id
PHONEPE_SALT_KEY=your_salt_key
PHONEPE_SALT_INDEX=1
PHONEPE_ENV=production       # Use 'sandbox' for testing

# ============================================
# FILESYSTEM
# ============================================
FILESYSTEM_DISK=local

# ============================================
# LOGGING
# ============================================
LOG_CHANNEL=daily            # Keeps logs per day
LOG_LEVEL=error              # Use 'debug' only locally
```

## Hostinger Shared Hosting Deployment

### Step 1 — Upload Files

Upload your project files to the hosting server. The **public folder** should map to your domain's `public_html`.

**Recommended structure on Hostinger:**
```
/home/u123456789/
├── domains/
│   └── vedantaplacementagency.in/
│       ├── public_html/          ← points to Laravel's /public folder
│       └── vedanta/              ← rest of Laravel project lives here
```

**How to do it:**
1. Upload the entire project to `/home/u123456789/vedanta/`
2. Move **all contents of the `public/` folder** to `public_html/`
3. Edit `public_html/index.php` — update the two paths:

```php
// Change from:
require __DIR__.'/../vendor/autoload.php';
$app = require_once __DIR__.'/../bootstrap/app.php';

// Change to:
require __DIR__.'/../vedanta/vendor/autoload.php';
$app = require_once __DIR__.'/../vedanta/bootstrap/app.php';
```

### Step 2 — Create Database

1. Go to **Hostinger → Databases → MySQL Databases**
2. Create a new database (e.g., `u123456789_vedanta`)
3. Create a database user and assign full permissions
4. Note down: **Host, Database Name, Username, Password**

### Step 3 — Configure .env

SSH into your server or use Hostinger's File Manager:
```bash
cd ~/vedanta
cp .env.example .env
nano .env  # Edit with your actual values
```

Fill in `DB_*` and `MAIL_*` values (see above).

### Step 4 — Run Setup Commands via SSH

Hostinger allows SSH access. Run these commands once:

```bash
cd ~/vedanta

# Install dependencies (production only)
composer install --no-dev --optimize-autoloader

# Generate app key
php artisan key:generate

# Run all database migrations
php artisan migrate --force

# Create storage symlink
php artisan storage:link

# Build frontend assets (if Node.js available)
npm install && npm run build

# Cache config, routes, and views for performance
php artisan config:cache
php artisan route:cache
php artisan view:cache
php artisan optimize
```

> ⚠️ **If Node.js is not available on the server**, build assets locally first (`npm run build`), then upload the generated `public/build/` folder to `public_html/build/`.

### Step 5 — File Permissions

```bash
chmod -R 755 ~/vedanta
chmod -R 775 ~/vedanta/storage
chmod -R 775 ~/vedanta/bootstrap/cache
```

### Step 6 — .htaccess (if needed)

In `public_html/.htaccess`, make sure this block exists:
```apache
<IfModule mod_rewrite.c>
    RewriteEngine On
    RewriteCond %{REQUEST_FILENAME} !-f
    RewriteCond %{REQUEST_FILENAME} !-d
    RewriteRule ^ index.php [L]
</IfModule>
```

## Email Setup on Hostinger

Hostinger provides free business email. Here's how to configure it:

### Step 1 — Create Email Account on Hostinger

1. Login to **Hostinger hPanel**
2. Go to **Emails → Email Accounts**
3. Click **Create Email Account**
4. Create: `info@vedantaplacementagency.in`
5. Set a strong password

### Step 2 — Get SMTP Details

From hPanel → **Emails → Email Accounts → Manage → Configuration**:

| Setting | Value |
|---|---|
| **SMTP Host** | `mail.vedantaplacementagency.in` |
| **SMTP Port** | `465` (SSL) or `587` (TLS) |
| **Username** | `info@vedantaplacementagency.in` |
| **Password** | *(your email password)* |
| **Encryption** | `SSL` (recommended) |

### Step 3 — Update .env

```dotenv
MAIL_MAILER=smtp
MAIL_SCHEME=ssl
MAIL_HOST=mail.vedantaplacementagency.in
MAIL_PORT=465
MAIL_USERNAME=info@vedantaplacementagency.in
MAIL_PASSWORD=YourEmailPasswordHere
MAIL_FROM_ADDRESS=info@vedantaplacementagency.in
MAIL_FROM_NAME="Vedanta Placement Agency"
```

### Step 4 — Test Email

After deployment, test email works via:
```bash
php artisan tinker --execute="Mail::raw('Test email', fn(\$m) => \$m->to('yourtest@gmail.com')->subject('Test'));"
```

## Scheduler & Cron Job Setup

This application uses Laravel's task scheduler for automated notifications. You must configure a **single Cron Job** on Hostinger to run every minute.

### What the Scheduler Runs

| Command | Schedule | Purpose |
|---|---|---|
| `invoices:calculate-late-fees` | Daily | Apply ₹300/day late fee to overdue service charge invoices & email candidates |
| `reminders:abandoned-registration` | Daily | Email candidates who registered 24h ago but haven't completed profile/payment |
| `reminders:interview` | Daily | Email candidates 24h before their scheduled interview |
| `reminders:lead-follow-ups` | Daily at 9:00 AM | Notify admin about leads due for follow-up today |

### Cron Job Setup on Hostinger

1. Login to **Hostinger hPanel**
2. Go to **Advanced → Cron Jobs**
3. Click **Create New Cron Job**
4. Set **Schedule** to **Every minute** (custom: `* * * * *`)
5. Set **Command** to:

```bash
/usr/bin/php /home/u123456789/vedanta/artisan schedule:run >> /dev/null 2>&1
```

> ⚠️ Replace `u123456789` with your actual Hostinger username.
> To find your PHP path on Hostinger, SSH in and run: `which php`

**Example with correct path:**
```bash
/usr/local/bin/php8.3 /home/u123456789/vedanta/artisan schedule:run >> /dev/null 2>&1
```

### Verify Cron is Working

After setting the cron, SSH in and run manually to check:
```bash
/usr/local/bin/php8.3 ~/vedanta/artisan schedule:run
```

You should see output like:
```
Running [reminders:abandoned-registration]
Running [invoices:calculate-late-fees]
...
```

### Queue Worker (Optional)

Emails use Laravel queues (`QUEUE_CONNECTION=database`). On shared hosting, a background worker is not always available. You have two options:

**Option A (Recommended for shared hosting):** Change to sync driver
```dotenv
QUEUE_CONNECTION=sync
```
This sends emails immediately (no queue), but can slow down response times slightly.

**Option B:** Run queue via cron (every 5 minutes)
```bash
*/5 * * * * /usr/local/bin/php8.3 /home/u123456789/vedanta/artisan queue:work --max-time=240 --stop-when-empty >> /dev/null 2>&1
```

## Automated Notifications Reference

### Emails Sent to Candidates

| Trigger | Mail Class | Template |
|---|---|---|
| Registration completed | `RegistrationSuccessMail` | `emails.candidate.registration_success` |
| Profile verified by admin | `ProfileApprovedMail` | `emails.candidate.profile_approved` |
| Job application status changed | `ApplicationStatusMail` | `emails.candidate.application_status` |
| Interview scheduled within 24h | `InterviewReminderMail` | `emails.candidate.interview_reminder` |
| Service charge invoice overdue | `LateFeeAlertMail` | `emails.candidate.late_fee_alert` |
| Registration incomplete after 24h | `AbandonedRegistrationMail` | `emails.candidate.abandoned_registration` |

### Emails Sent to Schools (Employers)

| Trigger | Mail Class | Template |
|---|---|---|
| Admin creates school account | `SchoolAccountCreatedMail` | `emails.employer.account_created` |
| Candidate profile forwarded | `CandidateForwardedMail` | `emails.employer.candidate_forwarded` |

### Admin Dashboard Notifications

| Trigger | Type | Where |
|---|---|---|
| Lead follow-up due today | DB Notification | Admin Dashboard Bell |
| Candidate profile verified | DB Notification | Candidate's Dashboard Bell |
| Service charge overdue | DB Notification | Candidate's Dashboard Bell |
